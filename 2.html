What is Full-Stack, Frontend & Backend?

Understand how a web app is built end-to-end — from the user’s browser to the database.

Quick Definition
  • Frontend — Everything the user sees and interacts with in the browser (UI).
  • Backend — The server side: APIs, business logic, databases, authentication.
  • Full-Stack — Works on both frontend and backend; connects the whole system.

1) Real-World Analogy

Frontend (Restaurant Menu)

The menu and table service you see and use.

HTML (structure) • CSS (design) • JS (interaction)

Backend (Kitchen)

Where orders are processed and food is prepared.

Server • APIs • Business Rules • Database

Full-Stack (Manager/Chef-Waiter)

Understands both menu & kitchen; ensures smooth delivery.

Bridges UI ↔ API ↔ DB

2) Request → Response Flow

User clicks “Login”
➜
Frontend sends POST /api/login
➜
Backend checks user, creates token
➜
DB returns user record
➜
Frontend shows dashboard ✅

3) Typical Tech Stack

LayerCommon TechWhat It Does
Frontend HTML, CSS, JavaScript, React/Vite UI pages, forms, navigation, validations, state, routing
Backend Node.js + Express, Python + Django/Flask, Java + Spring APIs, auth (JWT), business logic, security, server-side rendering
Database MongoDB, PostgreSQL, MySQL Store and query persistent data
DevOps Git/GitHub, CI/CD, Docker, Cloud Deploy, scale, monitor

4) Mini Code Glimpse

Frontend Submit a form:

// JavaScript (fetch)
fetch("/api/hello", { method: "GET" })
  .then(res => res.json())
  .then(data => console.log(data.message));

Backend Simple API (Node + Express):

// server.js
const express = require("express");
const app = express();
app.get("/api/hello", (req, res) => res.json({ message: "Hello from backend!" }));
app.listen(3000);

6) Quick Q&A

Q: Can a frontend app work without a backend?
A: For static sites, yes. For dynamic data/login, it needs an API (backend).

Q: Is database part of backend?
A: Commonly yes; backend talks to the DB.

Q: What makes a developer full-stack?
A: Comfortable building UI + APIs + working with databases and deployments.
